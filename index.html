01 — Stack técnico

Full Stack: React + Backend

Lo esperable en una entrevista junior no es que domines todo, es que expliques con precisión los conceptos base y por qué se usan.

Frontend

React — lo esencial

  • Componentes y props: unidades reutilizables de UI, reciben datos de solo lectura.
  • State (useState): datos que cambian y disparan un re-render.
  • useEffect: efectos secundarios (fetch, suscripciones), se ejecuta tras el render.
  • Virtual DOM: React compara versiones y actualiza solo lo necesario en el DOM real.
  • Levantar estado (lifting state up): compartir datos entre componentes hermanos vía el padre.
  • Keys en listas: ayudan a React a identificar qué elemento cambió.
Backend

API REST — lo esencial

  • GET / POST / PUT / DELETE: leer, crear, reemplazar, eliminar recursos.
  • Status codes: 200 ok, 201 creado, 400 mala petición, 401 no autenticado, 403 sin permiso, 404 no existe, 500 error servidor.
  • JSON: formato estándar de intercambio de datos.
  • Autenticación: JWT (token) vs sesiones con cookie.
  • Middleware: función que se ejecuta entre la petición y la respuesta (ej. validar token).
Ejemplo mental

Cómo viaja un dato de React al servidor

// 1. El componente pide datos al backend
async function cargarUsuarios() {
  const res = await fetch("/api/usuarios");
  const data = await res.json();
  setUsuarios(data); // 2. actualiza el state
} // 3. React re-renderiza la lista automáticamente
Herramientas

Git & control de versiones

  • commit / push / pull: guardar, subir y traer cambios.
  • branch: línea de desarrollo aislada (ej. feature/login).
  • merge vs rebase: unir historiales de dos formas distintas.
  • pull request: pedir revisión antes de integrar cambios.
Conceptos transversales

Que no se te olviden

  • CORS: por qué el navegador bloquea peticiones entre dominios distintos.
  • Variables de entorno (.env): nunca subir credenciales al repo.
  • Deploy: diferencia entre ambiente local, staging y producción.
  • Responsive design: que la UI funcione en distintos tamaños de pantalla.
02 — Python

Python para automatización y datos

Aunque el rol es full stack, Python suele aparecer para scripts, automatización, análisis y todo lo relacionado a datos — algo que ya trabajaste en tus notebooks de ML.

Fundamentos

Lo que puede preguntarte

  • Listas vs tuplas vs diccionarios: mutabilidad y cuándo usar cada una.
  • List comprehension: forma compacta de construir listas.
  • Manejo de excepciones con try / except.
  • Funciones, argumentos por defecto, *args / **kwargs.
  • POO básica: clases, atributos, métodos, herencia simple.
Aplicado

Librerías que puedes mencionar

  • pandas: manipulación de datos tabulares (ya lo usaste en tu dashboard de LoL).
  • requests: consumir APIs desde Python.
  • scikit-learn: lo que usaste para Gradient Boosting / K-Means.
  • Flask / FastAPI: frameworks livianos para levantar una API rápido.

03 — Bases de datos

SQL y modelado de datos

SQL básico

Consultas que deberías poder escribir a mano

SELECT nombre, total
FROM pedidos
JOIN clientes ON pedidos.cliente_id = clientes.id
WHERE total > 10000
ORDER BY total DESC;
Conceptos

Preguntas típicas

  • SQL vs NoSQL: estructura relacional con esquema fijo vs documentos flexibles (ej. MongoDB).
  • Normalización: reducir redundancia separando datos en tablas relacionadas.
  • Primary key / Foreign key: identificador único y relación entre tablas.
  • Índices: aceleran búsquedas a costa de más espacio y escrituras más lentas.
04 — Seguridad de la Información

SGSI — lo que probablemente te pregunten

Esta es la parte que más se sale de lo puramente técnico y la que menos suelen preparar los postulantes. Dominarla te puede diferenciar.

Base conceptual

La tríada CIA

  • Confidencialidad: solo quien está autorizado accede a la información.
  • Integridad: los datos no se alteran de forma no autorizada.
  • Disponibilidad: la información está accesible cuando se necesita.
Marco de referencia

SGSI / ISO 27001

  • SGSI = Sistema de Gestión de Seguridad de la Información, un conjunto de políticas y controles.
  • ISO 27001 es el estándar internacional más común para certificar un SGSI.
  • Ciclo PDCA (Plan-Do-Check-Act): planificar, implementar, revisar, mejorar.
  • Gestión de riesgos: identificar, evaluar y tratar riesgos de seguridad.
Práctica diaria

Buenas prácticas que puedes nombrar

  • No subir credenciales ni .env a repositorios.
  • Principio de mínimo privilegio: acceso solo a lo necesario.
  • Autenticación multifactor (MFA).
  • Reportar incidentes de seguridad apenas se detectan, no ocultarlos.
Respuesta sugerida

Si preguntan "¿qué sabes de SGSI?"

Puedes responder algo como: "Entiendo el SGSI como el marco de políticas que protege confidencialidad, integridad y disponibilidad de la información, alineado normalmente a ISO 27001. En mis proyectos ya aplico buenas prácticas básicas, como no exponer credenciales y usar variables de entorno, y estoy en posición de aprender los procedimientos específicos de la empresa."

08 — Preguntas frecuentes

Preguntas que puedes esperar

Usa el método STAR: Situación, Tarea, Acción, Resultado. Ejemplo: cuando depuraste el modelo Transformer que daba respuestas de baja calidad — el problema (Situación/Tarea), lo que hiciste (warmup, beam search, label smoothing — Acción), y la mejora obtenida (Resultado).
Di lo que sí sabes que se relaciona, y sé honesto: "No lo he trabajado en profundidad, pero entiendo que se relaciona con X, y me gustaría profundizar en eso." Es una entrevista para practicante — se espera que no sepas todo.
Estructura: problema → decisión técnica → resultado. No digas "usé React y Node", di "necesitaba que el dashboard reaccionara en tiempo real al cambiar de modo, así que separé el estado del tema en un contexto global y todos los gráficos de Chart.js se re-pintan solos".
Ejemplos: "¿Cómo es un día típico del equipo de tecnología?", "¿Qué herramientas de IA usa el equipo actualmente?", "¿Cómo se estructura el SGSI en la práctica del día a día?"
Sé específico sobre tu parte: "En el equipo trabajamos con roles divididos, yo lideré el desarrollo del pipeline de ML y la parte de visualización, coordinando con mis compañeros en el análisis de datos." Muestra que puedes trabajar en equipo sin diluir tu aporte individual.
